# Agent Flow

The agent flow comprises a series of structured steps wrapped around every call to the GitHub Copilot CLI. Before an agent is prompted, mandatory pre-agent checks are executed via scripting to ensure all necessary assets are correctly positioned, formatted, and consistent. These mechanical checks must be individual and modular rather than combined into a single compound check; for example, separate scripts should verify the file structure, document format accuracy, and cross-referencing consistency. Each check must generate clear, codified output so that if a failure occurs, the system can branch to a specific repair step before looping back to continue the process.The agent execution phase utilizes a preformatted prompt template. Placeholders within this template are mechanically replaced by a script immediately before the prompt is pushed to the agent. Once the agent completes its task, it returns a status which is then mechanically verified to ensure it is valid and contains the required information. Based on this verified status, a scripted decision determines the subsequent action. Finally, if all criteria are met, the flow concludes with a mechanical git sequence involving add, commit, and push operations.
