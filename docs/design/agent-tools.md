# Agent Tools

I want to abstract out all the mechanical aspects of the requirement specification, that an AI agent doesn't really need to know about in order to write a requirement. The thinking is, is that the details of the timestamp or the ruid will cause the agent to burn additional tokens and focus trying to figure things out. When really what we want it to do is focus on the requirement itself and the context and state of that requirement. So I'm thinking we need a set of tools on the command line for creating a child requirement from a requirement ruid. Creating a sibling requirement from a requirement are ruid, etc. This command line tool can insert things like timestamp, figure out what the RUID is. It can also, do mechanical validation of what the agent is creating. This same tool can also fetch requirements. Anything that's mechanical or can be mechanically validated or mechanically inferred doesn't need to go to the agent. We want it to focus on the requirement itself. Given this goal, please propose a set of. Commands. Likely parameters and arguments to the same CLI. That an agent can use to fetch and create and modify requirements.
